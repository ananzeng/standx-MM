# Add Rule

使用者描述一個想要加入的規範或偏好，分析並寫入最適合的設定檔。

## 分析邏輯

根據以下標準判斷放哪裡：

**放 `~/.claude/CLAUDE.md`（全域，每次對話都載入）**
- 語言偏好、回覆風格
- 開發環境設定（venv、套件管理工具等）
- Git commit 風格
- 通用程式碼風格（跨語言、跨專案都適用）

**放 `~/.claude/rules/<topic>.md`（全域 rule，可加路徑條件）**
- 針對特定檔案類型才適用的規則（加 `paths:` frontmatter）
- 可以歸類成主題模組的規範（如 python-deps、security、testing）
- 不需要每次對話都載入，只在相關情境才需要

**放 `~/.claude/skills/<name>/SKILL.md`（手動觸發的工作流程）**
- 多步驟的工作流程（需要查詢、執行、修改多個檔案）
- 需要明確觸發才執行的任務
- 有輸入參數的任務（如「幫我建立 X」）

**修改 `~/.claude/settings.json` 的 hooks（工具執行前後的自動行為）**
- 工具執行前後要自動執行的 shell 指令（PreToolUse / PostToolUse）
- 不是「規範」或「工作流程」，而是「副作用」或「通知」
- 範例：呼叫 Skill 後顯示通知、Edit 前自動備份、Bash 前印出提示

## 執行步驟

1. 讀取使用者描述的規範（來自 $ARGUMENTS 或對話）
2. 依照上述邏輯判斷放哪裡，並說明理由
3. 讀取目標檔案現有內容（如果存在）
4. 將規範整合寫入目標檔案（不覆蓋現有內容，加在適當位置）
5. 告知使用者寫入了哪個檔案的哪個位置

## 注意事項
- 如果規範可以放在現有 rule 檔案裡，優先整合進去，不新開檔案
- 如果是新的主題，才建立新的 rule 檔案
- 寫入前先確認規範不與現有設定衝突或重複
